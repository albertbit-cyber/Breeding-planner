# Beta Onboarding Flow Plan

Date: 2026-05-20

## Flow

1. Invite beta user.
2. Confirm role: breeder, lab, marketplace/admin.
3. Provide staging/beta URL.
4. User creates or receives account.
5. User completes first key workflow.
6. User submits feedback through support channel.

## First Workflows

- Breeder: add animal, create lab order, download certificate.
- Lab: receive order, update status, enter results.
- Marketplace: create listing, upload media, report/block test flow.

## Guardrails

- Mark beta clearly.
- Explain whether beta data can be reset.
- Keep support and feedback path visible.
