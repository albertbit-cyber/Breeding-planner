# Public Beta Runtime Rules Plan

Date: 2026-05-20

## Beta Rules

- Start invite-only.
- Use staging or beta-specific infrastructure, not production customer infrastructure, until launch approval.
- Keep feature flags for risky workflows.
- Keep feedback/support channel active.
- Make data-loss risk clear to testers if beta data may be reset.

## Entry Criteria

- Staging deployment complete.
- Staging smoke tests pass.
- Staging live E2E passes.
- Monitoring active.
- Backup and rollback tested.
- Legal/compliance checklist reviewed.

## Exit Criteria

- No critical workflow blockers.
- No unresolved high-severity security issues.
- Support process proven.
- Production release gate approved.
